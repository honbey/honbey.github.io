各类硬件速度

John von Neumann

  • CPU 一个时钟周期 1 纳秒上下，内存算是比较接近 CPU 的
  • 内存读数据：纳秒级别，10 ~ 80 纳秒
  • 千兆网卡读数据：微妙级别。1 微秒 = 1,000 纳秒，网卡比内存慢了 1,000 倍
  • 磁盘读数据：毫秒级别。1 毫秒 = 1,000,000 纳秒，硬盘比内存慢了 1 ~ 10,000 倍

并行和并发：Concurrency/Parallel 。 同步/异步，阻塞/非阻塞：Synchronous/Asynchronous 。 输入/输出，I/O

DMA

Direct Memory Access 直接存储器访问。它允许不同速度的硬件装置来沟通，而不要需 CPU 的参与。否则， CPU 需要从来源把每一分段的资料复制到暂存器（缓冲区），然后把他们再次写回到新的地方，在这个时间里， CPU 对于其他的工作来说就无法使用。

中央处理器 - CPU

制程

比如说 7nm 工艺制程，越小，精细度越高，就能在同样大小的尺寸中集成更多的元器件，功耗和发热一般就控制的更好。例如麒麟 9000 ，5nm工艺制程，集成的晶体管 153 亿个。

架构

CPU 所使用的指令集，例如 x86(Intel, AMD) 使用的是 CISC 复杂指令集，而 arm(Kirin 9000, M1) 使用的是 RISC 精简指令集。指令就是指挥计算机工作的指示和命令，程序就是一系列按一定顺序排列的指令，执行程序的过程就是计算机的工作过程。指令集就是 CPU 中用来计算和控制计算机系统的一套指令的集合，而每一种新型的 CPU 在设计时就规定了一系列与其他硬件电路相配合的指令系统。

基带

处理移动网络信号的设备，部分手机 CPU 集成了基带。

NPU

神经网络处理单元，通俗的说就是 AI 处理器。

屏幕 - Screen

分辨率

屏幕能够显示的像素个数，如 1920 x 1080 1080P FHD(Full High Definition) 就是能显示 2073600 个像素。

像素密度 - PPI

Pixels Per Inch，顾名思义，就是每英寸（1inch2 面积）能够显示的像素个数。值越高，显示的越细腻，效果越好。

色域

色彩显示范围。任何显示屏都不能完全显示出真实世界的所有色彩，色域标准就是用于衡量显示设备对于色彩显示范围的大小，色域越广越好。手机屏幕主流的是 DCI-P3，电脑显示器主流的是 sRGB。

刷新率

屏幕每秒的刷新次数，越高说明显示的越流畅。

采样率

触摸屏的指标，代表了触控操作的识别效率。简单理解为用手在手机上画线，采样率越高，线越连贯。

峰值亮度

屏幕能达到的最高亮度，单位 nit (1nit=1 cd/m²)。一般越高在强光下显示的越清楚，户外玩手机的比较注重这个参数。

亮度是指发光体（反光体）表面发光（反光）强弱的物理量。

对比度

画面明暗的差异程度，对比度越高，手机画面层次越多，细节越好。
